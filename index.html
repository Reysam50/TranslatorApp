Full Project Description

PolyLingo Translator is a web application designed to provide language translation capabilities through a modern browser-based interface. The system allows users to enter text, select source and target languages, and receive translated results in real time.

Unlike traditional translation projects that directly consume public APIs, this project deploys a self-hosted LibreTranslate service on a remote Linux VPS and uses a Flask backend as an intermediary layer between the frontend and translation engine. This architecture improves flexibility, allows future integration of multiple translation providers, and demonstrates practical API consumption and backend engineering concepts.

The application automatically retrieves supported languages from the translation server, dynamically populates language selection dropdowns, and provides additional usability features such as text-to-speech output, translation result copying, loading indicators, and responsive design.

This project was also used as a learning exercise to gain hands-on experience with:

  • Flask web development
  • HTTP requests and responses
  • REST API consumption
  • JSON data handling
  • Linux server deployment
  • Reverse proxy configuration
  • Client-server architecture
  • Frontend-backend communication

    Key Features

    Language Translation

    Translate text between multiple supported languages using a self-hosted translation engine.

    Dynamic Language Loading

    Supported languages are automatically retrieved from the translation server and populated into dropdown menus.

    Copy Translation

    Users can instantly copy translated text to the clipboard.

    Text-to-Speech

    Translated text can be spoken aloud using the browser's built-in speech synthesis capabilities.

    The application adapts to different screen sizes including desktops, tablets, and mobile devices.

    Loading Indicators

    Visual feedback informs users when translation requests are being processed.

    API Endpoints

    Home Page

    GET /

    Purpose:

    Render the main application interface.

    Translate

    POST /translate

    Purpose:

    Receive text and language selections from users and return translated content.

    Request Example:

    {
    "text": "Hello",
    "source": "en",
    "target": "fr"
    }

    Response Example:

    {
    "translatedText": "Bonjour"
    }

    Languages

    GET /languages

    Purpose:

    Retrieve supported languages from the translation service.

    Response Example:

    [
    {
    "code": "en",
    "name": "English"
    },
    {
    "code": "fr",
    "name": "French"
    }
    ]
